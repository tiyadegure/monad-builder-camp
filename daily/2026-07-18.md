### 1. GitHub 探索日志（Moss / Solidity / Hardhat 对比）

学习的不是单个项目，而是"怎么读一个真实开源项目"：

- **Moss**：monorepo（pnpm workspace），`packages/{core,erc,protocols,simulator,system,mcp-server}`，文档在 `docs/`，有 `adr/` 架构决策记录
- **Hardhat**：同样是 monorepo，但包更多（25+），`packages/hardhat-*` 系列，是成熟工具的工程化范本
- **argotorg/solidity**：聚焦形式化验证，结构更小

**最感兴趣的 PR**：Moss 的 **PR #84（Uniswap V4 adapter）**——在 Monad 上封装 Uniswap V4，说明 Moss 的 Capability 抽象足够通用，落地后 Agent 能自动做 Unad 上的 Uniswap V4 swap。

**学习收获**：monorepo + 清晰 package 边界 + ADR 是专业项目的标志；判断一个 PR 处于什么阶段，看它是"新协议包"还是"文档/修复"；stars 数不是项目活跃度的唯一指标，PR 和 Issue 的近期活动更重要。

### 关键认知

- **教程内容要来自真实运行，不是想象。** 把 Moss 跑通后，Receipt 字段、合约地址、报价数字都是一手证据，比任何概括都扎实。
- **Kami 美学 = 克制。** 单一墨蓝强调色、衬线、细线边框，不用多彩色块和 emoji。blog 自带 Kami 设计语言（global.css 是 parchment + ink-blue + serif），教程样式要与之对齐。
- **Moss 安全模型核心**：先验证（Receipt 与意图一致），后签名。教程反复落到这一点。
- **开源贡献的低成本入口是文档**，尤其中文文档。不需要写核心代码也能有效参与。
