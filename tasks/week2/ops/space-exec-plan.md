# 活动执行预案：用 Moss 在 Monad 上接一个协议（Space 实战夜）

## 时间排期

活动定档后倒推 3 周准备期（以活动日 D 为基准）：

| 阶段 | 时间 | 关键动作 |
|------|------|----------|
| 筹备启动 | D-21 | 确定主题与嘉宾、锁定活动日、起草内容设计（Run of Show） |
| 内容打磨 | D-14 | 嘉宾对齐分享大纲、准备真实案例素材（aPriori adapter PR #104 截图/录屏）、互动环节题库定稿 |
| 宣传启动 | D-10 | 发布预告帖、建活动页、开启报名/提醒 |
| 宣传加码 | D-5 | 发布嘉宾介绍、抛出预热问题、社群转发 |
| 最终确认 | D-2 | 全流程彩排（主持人+嘉宾 30 分钟）、检查推流/录音/材料 |
| 活动当天 | D | 按 Run of Show 执行 |
| 收尾沉淀 | D+2 | 发布回顾帖、整理录音/文字稿、跟进 CTA（star / fork / 认领任务） |

## 宣传计划

节奏：低频铺垫 → 高频提醒，避免过度打扰。

- **D-10 预告**：一条主题帖讲清"这场 Space 你能带走什么"（不说空话，直接给案例截图），附活动时间与收听链接。
- **D-7 嘉宾介绍**：分 1–2 条短帖介绍嘉宾与他们会讲的真东西（踩坑案例 > 概念）。
- **D-5 预热问题**：在社群/评论区抛 Run of Show 里的核心问题之一（如"接协议=写合约吗？"），引发讨论并导流到活动。
- **D-2 倒计时**：明确时间、平台（Twitter Space）、如何设提醒。
- **D 当天**：活动前 2 小时再提醒一次；活动中主持人实时发关键结论片段。
- **D+2 回顾**：图文回顾 + 录音/文字稿链接 + CTA 重申。

渠道：Twitter/X 主页 + Monad / Moss 相关社群（Discord / Telegram）+ 学习仓库 `tech-track/` 作自学留档。

## 人员分工

| 角色 | 人数 | 职责 |
|------|------|------|
| 总负责人 | 1 | 整体排期、对外确认、风险兜底 |
| 主持人（Moderator） | 1 | 开场串场、控节奏、念互动答案、收尾 CTA |
| 分享嘉宾 | 1–2 | 主题分享①（Agent 框架）、分享②（aPriori 真实案例） |
| 内容/素材 | 1 | 准备案例截图、录屏、题库、回顾图文 |
| 宣传 | 1 | 排期发帖、社群转发、倒计时提醒 |
| 技术支持 | 1 | 活动前彩排、当天推流/录音/网络兜底 |

（30–100 人规模下，以上角色可由 2–3 人兼任，无需满编。）

## 执行流程（活动当天）

1. **T-30min**：技术到位，主持人+嘉宾进入直播间，确认音频/录屏。
2. **T-5min**：发布"马上开始"提醒，测试连麦通道。
3. **00:00 开场**（5min）：主持人自我介绍、嘉宾介绍、抛出今晚一句话目标。
4. **05–30min 主题分享**：分享① + 分享②，按 Run of Show 时间轴推进。
5. **30–40min 互动**：主持人抛 3 个"一句话需求→猜函数"，实时念答案纠偏。
6. **40–52min 圆桌**：围绕 5 个核心问题连麦讨论。
7. **52–58min 收尾 CTA**：三级行动（star / fork / 认领任务）+ 自学手册指向。
8. **58–60min Q&A**：自由提问，主持人兜底。
9. **结束后**：停止录音、保存素材、技术发回顾预告。

## 风险预案

| 风险 | 表现 | 应对 |
|------|------|------|
| 嘉宾临时缺席 | 分享②无人讲 | 总负责人提前备好案例录屏，改为"主持人播片+解说"；或延期 1 周 |
| 听众少（<30） | 互动冷场 | 预设 3–5 名"托"（社群伙伴）主动在聊天区接龙，保证互动不空 |
| 音频/网络故障 | 听不见、掉线 | 技术支持 T-30 彩排；准备备用账号；掉线即重开并播报 |
| 讨论跑题 | 超时、节奏乱 | 主持人严格按时间轴控场，每个问题限时 2–3 分钟 |
| 案例素材出错 | 截图/函数名不对 | 内容岗 D-2 彩排时逐页核对 PR #104 与链上 selector |
| 互动答案争议 | 听众函数签名不同 | 主持人引导"以链上验证为准"，现场用 4byte.directory 收口 |

## 预期目标

- **参与**：30–100 人实时收听，互动环节聊天区有效发言 ≥ 15 条。
- **认知**：听众能说清"接协议≠写合约，而是封装已有函数"，并知道用 `eth_call`/4byte.directory 自查。
- **转化（CTA）**：活动后一周内 PR #104 新增 ≥ 5 个 star；≥ 3 人 fork 跑通 Moss；≥ 1 人认领 Moss 小任务（文档/FAQ 或新 adapter）。
- **沉淀**：产出回顾图文 + 文字稿，作为 `tech-track/` 之外的公开自学材料。

（以上目标为预案设定值，非真实数据，活动未实际举办。）
