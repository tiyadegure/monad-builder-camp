关键代码（真实片段）

// src/abis/apriori.ts —— 链上验证注释（ADR 0007）
// aprMON is an EIP-7702 delegated token; logic in 0x29fcb43b46531bca003ddc8fcb67ffe91900c762
// deposit 0x6e553f65 / requestRedeem 0x107703ab / redeem 0x492e47d2

// src/adapter.ts —— stake Capability（payable + msg.value）
@Capability({ verb: "stake", params: stakeParams, receipt: "stakeReceipt", risk: ["fundOut","priceImpact"] })
async stake(params) {
  const amountBase = parseUnits(params.amount, 18);
  return [this.aprMon.deposit([amountBase, params.receiver], { value: amountBase })];
}

Known Issues（诚实标注）

  • 主网 stake 模拟 revert：deposit(uint256,address) 用真实余额账户 + value 调用仍 revert。函数存在（selector 验证），卡业务逻辑层（receiver 形式 / 最小量 / 白名单待确认）。不影响交易构建正确性。
  • redeem 数组参数：Moss 当前 param 类型无数组变体，claim 暂只接受单 requestId，批量领取待框架支持后补。
  • unstake/claim 主网 simulate：requestRedeem/redeem 未端到端验证（需真实 aprMON 持仓与已过 unbonding epoch 的 request）。
