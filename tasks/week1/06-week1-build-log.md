### 遇到的问题与修复

**问题 1：测试网钱包没钱**
- 现象：forge create 报 "insufficient balance"
- 原因：新建的钱包地址和之前质押用的不是同一个
- 修复：用有余额的钱包私钥重新部署

**问题 2：getMessages() 返回数据解码失败**
- 现象：cast call 指定错误的返回类型，报 "buffer overrun"
- 原因：struct 数组的 ABI 解码不能简单指定返回类型
- 修复：用 raw call 获取 hex 数据，手动解码字符串内容

### 对 Monad / Web3 理解的变化

**之前：** 以为链上产品就是"把数据库换成区块链"。

**现在理解：** 链上产品的核心差异不是存储方式，而是**信任模型**。普通产品是"你得信我"，链上产品是"不用信我，代码都在链上"。但这个信任是有代价的——gas 成本、操作不可逆、用户体验门槛更高。Monad 的价值在于降低这个代价：低 gas 让小额交互可行，快速确认让体验接近 Web2。

另一个变化是理解了 **AI + 链上** 的组合价值。AI 帮我快速生成合约初稿，但最终的安全判断、设计取舍必须由人来做。这个"AI 生成 + 人工审查"的模式可能是 Web3 开发的常态。

## 最重要的 3 个学习收获

1. **链上产品的核心是信任模型**，不是技术栈。"不用信我，代码都在链上"比"更快更便宜"更能解释为什么需要区块链。

2. **AI 生成代码必须人工审查**。AI 能写出 90% 正确的合约，但可见性选择、函数取舍、安全边界这些判断只能由人来做。

3. **完整链路比单点知识更重要**。从 Prompt → 代码 → 编译 → 部署 → 交互 → 浏览器验证，每个环节都有坑，走一遍完整流程比看十篇教程更有用。
